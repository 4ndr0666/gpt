# Media Prompt Director

This skill provides an automated pipeline for generating, checking, and hardening structural asset descriptions optimized for execution on Google's specialized creative models.

---

## 1. Nano Banana Pro Variable Configuration Engine

The `gemini-3-pro-image-preview` architecture replaces standard descriptive text blocks with an isolated multi-variable array grid. To prevent visual distortion or text fragment artifacting, use the formula below:

```text
[VARIABLES]
SUBJECT    = "Detailed visual character profile, specific attire components, and structural material properties."
SETTING    = "Spatial orientation coordinates, background density controls, and macro environment boundaries."
LIGHTING   = "Explicit lighting equipment types, intensity maps, illumination directions, and bounce characteristics."
STYLE_TEXT = "Camera kit settings, exact lens profiles, aperture numbers, and double-quote encapsulated text strings."

```

### Critical Rules

* **Text Boundary Protection**: Text parameters must be grouped together in literal double quotes followed by styling arrays (e.g., `"TEXT_STRING" rendered cleanly in a bold, metallic-gold sans-serif font`).
* **Real-World Optical Maps**: Emulate physical production gear setups directly (e.g., `Shot on 100mm Macro lens at f/4 with simulated focus-stacking pass for crisp detail`).
* **Environmental Exclusion Filtering**: Avoid using negative prompt patterns (`no traffic`, `don't show text`). Use descriptive environmental phrases to specify what should be missing (e.g., `a desolate landscape completely devoid of man-made structures`).
* **Grounding Core Engine**: For charts, maps, or real-world data data layouts, trigger live info pulls using this strict sequence: `[Search for current weather/metrics in location] + [Analytical Mapping] + [Visual Translation Grid]`.

## 2. Google Veo 3.1 Cinematic Video Formula

Google Veo 3.1 generates video clips using explicit sequential weights. To maintain structural balance over extended generations, the total prompt configuration should stay between 75-150 words and use the structural blocks below in this exact order:

### Part 1: Cinematography & Camera Mechanics

Define the explicit optical lens adjustments and camera movement vector right away at the start of the prompt.

* *Production Targets*: `Eye-level follow tracking shot on 35mm film with natural grain`, `Low-angle tracking shot`, `Dutch angle slow push-in`, or `Crane shot slowly descending`.

### Part 2: Primary Subject Profile

Isolate individual character traits, uniform materials, clothing styles, and facial expressions into a clean description block.

### Part 3: Kinetic Motion & Force Physics

To prevent fluid or dreamlike artifacts across frames, all movements must be driven by active force physics tokens.

* *Production Targets*: Use specific momentum words (`slam`, `thrust`, `shattered`, `shimmering`, `piercing`). Avoid passive verbs.

### Part 4: Contextual Setting & Atmosphere

Specify environmental factors, time blocks, weather levels, and atmospheric particle effects (`heavy volumetric dust clouds at twilight`, `refractive rain streaks down glass panes`).

### Part 5: Soundstage & Audio Orchestration

Decouple dialog strings, foley effects, and environmental noise maps into an explicit script array to keep them from getting mixed up with standard visual prompt elements:

* `Character says: "[exact text]" (no subtitles)`
* `Audio: [explicit description of environmental background ambiance]`
* `Sound effects: [tactical, synchronized physical sound keys]`
